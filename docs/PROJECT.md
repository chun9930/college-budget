# PROJECT.md

## 1. 프로젝트 개요

이 프로젝트는 대학생이 지금 소비를 해도 되는지 즉시 판단할 수 있는 React + Vite 기반 지출 관리 웹 UI이다.

핵심 원칙:
- 소비 기록보다 소비 판단을 우선한다.
- 한 화면에는 하나의 목적만 둔다.
- 모바일에서 직관적으로 사용 가능해야 한다.
- 복잡한 분석보다 빠른 판단을 우선한다.

프로젝트의 성격:
- 이 프로젝트는 지출 관리 앱이 아니라 소비 판단 도구이다.

---

## 2. 기술 및 구조

- React + Vite
- JavaScript
- localStorage 기반 상태 관리
- API 없음
- 백엔드 없음
- Hash Router 사용

추가 기준:
- 화면별 역할은 `pages`에서 관리한다.
- 재사용 UI는 `components`에서 관리한다.
- 계산과 저장 로직은 `lib`에서 관리한다.
- 로컬 저장소는 `src/lib/storage.js`를 통해서만 접근한다.

---

## 3. 라우팅 구조

- `/#/` → 홈

## 6. 회원가입 / 로그인 정책

이 프로젝트의 회원가입과 로그인은 실제 서버 인증이 아니라 localStorage 기반 mock auth로 구현한다.

목적은 실제 보안 인증이 아니라 프론트엔드 사용자 흐름을 구현하는 것이다.

### 6.1 페이지

#### `/#/login`
- 이메일 입력
- 비밀번호 입력
- 로그인 버튼
- 회원가입 페이지 이동

#### `/#/signup`
- 사용자 이름 입력
- 이메일 입력
- 비밀번호 입력
- 비밀번호 확인 입력
- 회원가입 버튼
- 로그인 페이지 이동

#### `/#/my-page`
- 사용자 이름 표시
- 이메일 표시
- 예산 요약
- 목표 요약
- 기능 토글 상태
- 데이터 초기화
- 로그아웃

### 6.2 localStorage 키

- `users`
- `loginState`
- `userProfile`

### 6.3 동작 기준

- 회원가입 시 `users`에 사용자 정보를 저장한다.
- 같은 이메일로 중복 가입할 수 없다.
- 로그인 시 `users`에서 이메일과 비밀번호가 일치하는지 확인한다.
- 로그인 성공 시 `loginState`와 `userProfile`을 저장한다.
- 로그아웃 시 `loginState`와 `userProfile`을 제거한다.
- 비밀번호 찾기, 이메일 인증, 서버 인증, 토큰 인증은 구현하지 않는다.

### 6.4 주의사항

- 이 기능은 실제 보안 인증이 아니다.
- 비밀번호는 실제 서비스 수준의 안전한 저장을 제공하지 않는다.
- 이 기능은 포트폴리오용 프론트엔드 흐름 구현을 위한 mock auth이다.

- 목표 설정 ON/OFF
- 기간별 저축 계산 ON/OFF

초기값:
- 모두 ON

#### 3) 목표 설정 카드
목표 설정 ON일 때만 표시한다.

- 목표 금액

- 현재 저축 금액

#### 4) 기간별 계산 카드
기간별 계산 ON일 때만 표시한다.

- 하루 필요 저축 금액
- 주간 필요 저축 금액
- 월 필요 저축 금액

## 8. 핵심 계산 로직

### 8.1 하루 예산

하루 예산 =

`(월 수입 또는 수동 예산 + 이월 금액 - 목표 저축 - 비상금 - 고정지출 - 누적 지출) / 남은 일수`

- 남은 일수는 오늘을 포함한다.
- 수동 예산 입력값이 있으면 수동 예산을 우선한다.

### 8.2 목표 설정 반영

- 목표 설정 OFF: 목표 금액 계산에서 제외한다.
- 목표 설정 ON: 목표 기간 기준 필요 저축 금액을 계산한 뒤 예산에서 차감한다.

### 8.3 기간별 저축 계산

필요 저축 =

`(목표 금액 - 현재 저축) / 남은 기간`

출력:
- 하루
- 주
- 월

### 8.4 이월 기능

- ON: 전날 남은 금액을 다음 날 예산에 추가한다.
- OFF: 이월하지 않는다.

### 8.5 경고 상태 기준

- 안전: 0~69%
- 주의: 70~89%
- 위험: 90~99%
- 초과: 100% 이상

### 8.6 초과 처리

- 예산은 변경하지 않는다.
- 알림 상태만 다시 계산한다.
