# PROJECT.md

## 1. 프로젝트 개요

이 프로젝트는 대학생을 위한 생활지출 관리 웹 서비스이다.

핵심 목표:
- "지금 이 소비를 해도 되는지"를 실시간으로 판단한다.
- 소비 기록보다 소비 판단에 초점을 둔다.

기존 가계부와의 차이:
- 소비 기록 중심: 가계부
- 소비 판단 중심: 본 프로젝트

추가 기준:
- UI는 직관적이고 단순해야 한다.
- 디지털 접근성이 낮은 사용자도 쉽게 사용할 수 있어야 한다.
- 모바일에서도 보기 편한 구조여야 한다.

- React (Vite)
- JavaScript
- HTML / CSS

제한:
- 백엔드 없음
- DB 없음
- 외부 API 없음

추가 기준:

## 4. 데이터 처리 방식

기본:
- 더미 데이터를 사용한다.

상태 유지:
- localStorage 사용 가능

규칙:
- localStorage 접근은 `lib`에서 관리한다.
- 저장 키는 명확하게 정의한다.
- 예산, 지출 기록, 고정지출, 템플릿, 경고 상태는 각각 분리해서 저장한다.

추가 기준:
- 데이터 구조는 단순하고 예측 가능해야 한다.
- 화면이 아니라 데이터 기준으로 상태를 관리한다.

로그인 기능은 구현 가능하다.

## 7. 핵심 로직

### 7.1 하루 예산 계산

- 하루 예산은 남은 금액을 남은 일수로 나누어 계산한다.
- 수동 예산 입력값이 있으면 수동 예산을 우선한다.
- 수동 예산이 없으면 월 수입 기준으로 자동 계산한다.
- 고정지출, 목표 저축, 비상금은 월 예산에서 먼저 제외한 뒤 하루 예산을 계산한다.
- 이월 기능이 켜져 있으면 전날 남은 금액을 오늘 예산에 반영한다.

하루 예산 계산식:
- 하루 사용 가능 금액 = (사용 가능 예산 + 이월 금액 - 고정지출 - 목표저축 - 비상금) / 남은 일수

### 7.2 남은 일수 계산

- 남은 일수는 오늘을 포함한다.
- 예시: 4월 10일이면 4월 10일부터 4월 30일까지를 남은 일수로 계산한다.

### 7.3 소비 판단

- 예산 이하이면 소비 가능으로 판단한다.
- 예산을 초과하면 경고를 표시한다.

상태 기준:
- 안전: 오늘 예산의 80% 미만 사용
- 주의: 오늘 예산의 80% 이상 사용
- 위험: 오늘 예산의 100% 이상 사용
- 초과: 오늘 예산을 넘김

### 7.4 경고 및 초과 알림

- 하루 기준과 월 기준 초과를 분리해서 계산한다.
- 초과 상태에서는 강한 경고 메시지를 표시한다.
- 사용자의 소비 패턴에 따라 알림 강도를 조절할 수 있다.

### 7.5 초과 후 재설정

- 초과 후 재설정은 예산 자체를 변경하지 않는다.
- 재설정은 알림 상태만 다시 계산한다.
- 사용자가 초과를 확인한 뒤에는 알림 배너를 해제할 수 있지만, 지출 데이터와 예산 계산 기준은 유지한다.

### 7.6 정기지출 자동 반영

- 정기지출은 지정한 결제일에 해당 월 지출로 자동 반영한다.
- 예시: 매월 5일 넷플릭스 17,000원은 매월 5일 지출 기록에 자동 추가한다.
- 정기지출은 사용자가 직접 등록, 수정, 삭제할 수 있어야 한다.

### 7.7 예외 처리

- 빈 값 처리
- 음수 처리
- NaN 처리
- 남은 일수 0 처리

추가 기준:
- 예외 상황에서는 계산이 중단되지 않도록 안전한 기본값을 사용한다.
- 오류 메시지는 사용자에게 이해하기 쉽게 표시한다.
