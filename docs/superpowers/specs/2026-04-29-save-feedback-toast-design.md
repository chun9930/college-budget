# 저장 피드백 토스트 디자인 명세

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 저장/적용 성공 여부를 각 페이지에서 짧은 토스트 메시지로 알려 사용자가 버튼 동작이 실제로 반영되었는지 즉시 확인할 수 있게 한다.

**Architecture:** 버튼 문구와 화면 구조는 유지하고, 성공 피드백만 공통 토스트로 분리한다. 토스트는 앱 내부 UI 상태로만 동작하며 localStorage에는 저장하지 않는다. 저장이 성공한 시점에만 표시되며, 각 페이지의 기존 계산과 저장 로직은 그대로 유지한다.

**Tech Stack:** React, React Router, localStorage, CSS

---

## 1. 배경

현재는 예산 설정, 지출 기록, 정기지출 관리 등에서 저장 버튼을 눌러도 사용자가 저장 성공 여부를 즉시 확인하기 어렵다.  
이 문제를 해결하기 위해 각 페이지의 버튼 이름은 그대로 두고, 저장 성공 시점에만 토스트 메시지를 보여주는 공통 피드백을 추가한다.

## 2. 범위

### 포함

- 예산 설정 페이지의 `저장하기` 성공 피드백
- 지출 기록 페이지의 일반 지출 저장/수정/삭제 후 피드백
- 지출 기록 페이지의 정기지출 저장/수정/삭제 후 피드백
- 정기지출 합계 불러오기처럼 form state만 바꾸는 동작에 대한 안내 메시지
- 홈이나 다른 페이지에서 공유 가능한 공통 토스트 UI

### 제외

- 버튼 텍스트 변경
- 계산 로직 변경
- 저장 데이터 구조 변경
- 브라우저 Notification API 사용
- 서버/백엔드/푸시 알림 추가
- 분석 페이지 알림 기능 추가

## 3. UX 원칙

- 저장이 성공한 후에만 토스트가 나타나야 한다.
- 입력만 하고 저장하지 않은 상태에서는 토스트가 나타나지 않아야 한다.
- 토스트는 짧고 명확해야 하며, 사용자의 현재 행동을 방해하지 않아야 한다.
- 각 페이지 버튼 이름은 그대로 유지한다.
- 성공 메시지는 페이지마다 다를 수 있지만 표현 방식은 공통이어야 한다.

## 4. 동작 규칙

### 4.1 예산 설정

- 사용자가 `저장하기`를 눌러 검증을 통과하면 토스트를 보여준다.
- 토스트 예시: `저장되었습니다`
- 정기지출 합계 불러오기 버튼은 저장이 아니라 form state 반영이므로 별도 안내 문구만 유지한다.

### 4.2 지출 기록

- 일반 지출 저장/수정/삭제가 성공하면 토스트를 보여준다.
- 정기지출 저장/수정/삭제가 성공하면 토스트를 보여준다.
- 최근 기록 자동 입력은 저장이 아니므로 토스트를 띄우지 않는다.

### 4.3 홈

- 홈은 판단/경고 중심 화면이므로 저장 성공 토스트의 주요 대상이 아니다.
- 다만 공통 토스트 컴포넌트는 재사용 가능하게 둔다.

## 5. UI 형태

- 토스트는 화면 위 또는 아래 한쪽에 고정된 작은 메시지로 표시한다.
- 짧은 텍스트와 닫기 없이 자동 소멸하는 방식으로 한다.
- 모바일에서는 화면 너비에 맞춰 한 줄 또는 두 줄로 자연스럽게 줄바꿈된다.
- 토스트가 여러 개 겹치지 않도록 한 번에 하나만 보이게 한다.

## 6. 대안 비교

### 추천안: 공통 토스트

- 장점: 화면을 거의 건드리지 않고 저장 성공을 분명히 알릴 수 있다.
- 장점: 예산 설정, 지출 기록, 정기지출 관리에 공통 적용 가능하다.
- 단점: 놓칠 수 있으므로 너무 짧게 사라지지 않도록 해야 한다.

### 대안 1: 인라인 성공 문구

- 장점: 사용자가 확인하기 쉽다.
- 단점: 입력 폼이 많은 화면에서 레이아웃이 흔들릴 수 있다.

### 대안 2: 확인 모달

- 장점: 가장 확실하다.
- 단점: 저장마다 흐름이 끊겨 현재 프로젝트 톤에 비해 무겁다.

## 7. 성공 기준

- 저장 버튼을 눌렀을 때만 토스트가 보인다.
- 입력만 한 상태에서는 토스트가 보이지 않는다.
- 각 페이지에서 저장 성공 여부를 즉시 알 수 있다.
- 기존 저장/수정/삭제, 계산, localStorage 구조는 유지된다.

## 8. 테스트 기준

- 예산 설정 저장 성공 시 토스트가 보이는지 확인한다.
- 일반 지출 저장/수정/삭제 성공 시 토스트가 보이는지 확인한다.
- 정기지출 저장/수정/삭제 성공 시 토스트가 보이는지 확인한다.
- 저장하지 않은 입력 변경에는 토스트가 뜨지 않는지 확인한다.
- 모바일에서 토스트가 화면 폭을 벗어나지 않는지 확인한다.
