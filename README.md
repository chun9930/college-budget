# college-budget-web

대학생이 오늘 소비 가능 여부를 빠르게 판단할 수 있도록 돕는 React + Vite 기반 지출 관리 웹 애플리케이션입니다.

## 1. 프로젝트 목표

- 오늘 사용 가능 금액을 즉시 확인한다.
- 소비 기록보다 소비 판단을 우선한다.
- 반복 입력을 줄이기 위해 템플릿을 제공한다.
- 소비 상태와 경고를 직관적으로 전달한다.
- 복잡한 분석이 아닌 이해 가능한 요약 정보를 제공한다.

## 2. 구현 범위

### MVP 기준

- 홈(오늘 현황 + 경고 배너)
- 예산 설정
- 지출 기록
- 분석 리포트

### 제외

- 로그인
- 마이페이지
- 별도 경고 센터(홈 배너로 처리)

## 3. 페이지 구조

Hash Router 기준 페이지 경로는 다음과 같습니다.

- `/#/` 홈
- `/#/budget-settings` 예산 설정
- `/#/expense-records` 지출 기록
- `/#/statistics` 분석

## 4. 데이터 저장 방식

모든 데이터는 `localStorage` 기반으로 관리합니다.

`src/lib/storage.js`를 통해서만 접근합니다.

사용 키:

- `monthlyIncome`
- `budgetSettings`
- `expenseRecords`
- `expenseTemplates`
- `alertState`
- `recurringExpenses`

## 5. 핵심 계산 로직

### 5.1 하루 예산 계산

하루 예산 =

`(월 수입 또는 수동 예산 + 이월 금액 - 목표 저축 - 비상금 - 고정지출 - 누적 지출) / 남은 일수`

- 남은 일수는 오늘을 포함합니다.
- 수동 예산 입력값이 있으면 수동 예산을 우선합니다.

### 5.2 소비 판단 기준

- 안전: 0~69%
- 주의: 70~89%
- 위험: 90~99%
- 초과: 100% 이상

### 5.3 초과 처리 방식

- 초과 후 재설정은 예산을 변경하지 않습니다.
- 알림 상태만 다시 계산합니다.

## 6. 지출 기록

### MVP 입력

- 금액
- 카테고리
- 결제수단

### 고급 옵션

- 지출 유형
- 메모
- 템플릿
- 정기지출 등록

## 7. 분석 리포트

- 월 지출 요약
- 카테고리 비중
- 가장 많이 쓴 항목
- 간단한 절약 힌트

## 8. UI 구조 원칙

- 한 화면에 하나의 목적만 둔다.
- 정보는 요약 → 핵심 → 상세 순서로 배치한다.
- 색상과 텍스트를 함께 사용해 상태를 전달한다.
- 버튼 최소 높이는 48px로 유지한다.
- 텍스트 최소 크기는 16px 이상을 기준으로 한다.
- 모바일 우선으로 설계한다.
- 로고는 교체 가능해야 하며, 레이아웃이 깨지지 않도록 고정 컨테이너를 사용한다.
