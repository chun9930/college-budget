# college-budget-web

대학생이 지금 소비를 해도 되는지 빠르게 판단할 수 있도록 돕는 React + Vite 기반 지출 관리 웹 애플리케이션이다.

## 1. 프로젝트 목표

- 오늘 사용 가능 금액을 즉시 확인한다.
- 소비 기록보다 소비 판단을 우선한다.
- 최근 기록 기반 자동 입력 보조로 빠른 지출 입력을 지원한다.
- 소비 상태와 경고를 직관적으로 전달한다.
- 복잡한 분석이 아닌 이해 가능한 요약 정보를 제공한다.

## 3. 페이지 구조

Hash Router 기준 경로는 다음과 같다.

- `/#/` -> 홈
- `/#/budget-settings` -> 예산 설정
- `/#/expense-records` -> 지출 기록
- `/#/statistics` -> 분석
- `/#/calendar` -> 캘린더
- `/#/login` -> 로그인
- `/#/signup` -> 회원가입
- `/#/my-page` -> 마이페이지

## 5. 회원가입 / 로그인 정책

회원가입과 로그인은 실제 서버 인증이 아니라 `localStorage` 기반 mock auth로 구현한다.

목적은 실제 보안 인증이 아니라 프론트엔드 사용자 흐름을 구현하는 것이다.

### 로그인 페이지

- 이메일 입력
- 비밀번호 입력
- 로그인 버튼
- 회원가입 페이지 이동

### 회원가입 페이지

- 사용자 이름 입력
- 이메일 입력
- 비밀번호 입력
- 비밀번호 확인 입력
- 회원가입 버튼
- 로그인 페이지 이동

### 마이페이지

- 사용자 이름 표시
- 이메일 표시
- 예산 요약
- 목표 요약
- 기능 토글 상태
- 데이터 초기화
- 로그아웃

### 동작 기준

- 회원가입 시 `users`에 사용자 정보를 저장한다.
- 같은 이메일로 중복 가입할 수 없다.
- 로그인 시 `users`에서 이메일과 비밀번호가 일치하는지 확인한다.
- 로그인 성공 시 `loginState`와 `userProfile`을 저장한다.
- 로그아웃 시 `loginState`와 `userProfile`을 제거한다.
- 비밀번호 찾기, 이메일 인증, 서버 인증, 토큰 인증은 구현하지 않는다.

## 6. 핵심 계산 로직

### 하루 예산

하루 예산 =

`(월 수입 또는 수동 예산 + 이월 금액 - 목표 저축 - 비상금 - 고정지출 - 누적 지출) / 남은 일수`

- 남은 일수는 오늘을 포함한다.
- 수동 예산 입력값이 있으면 수동 예산을 우선한다.

### 목표 설정 반영

- 목표 설정 OFF: 목표 금액 계산에서 제외한다.
- 목표 설정 ON: 목표 기간 기준 필요 저축 금액을 계산한 뒤 예산에서 차감한다.

### 기간별 저축 계산

필요 저축 =

`(목표 금액 - 현재 저축) / 남은 기간`

출력:

- 하루
- 주
- 월

### 이월 기능

- ON: 전날 남은 금액을 다음 날 예산에 추가한다.
- OFF: 이월하지 않는다.

### 경고 상태 기준

- 안전: 0~69%
- 주의: 70~89%
- 위험: 90~99%
- 초과: 100% 이상

### 초과 처리

- 예산은 변경하지 않는다.
- 알림 상태만 다시 계산한다.

## 7. 지출 기록

### MVP 입력

- 금액
- 카테고리
- 결제수단

### 고급 옵션

- 지출 유형
- 메모
- 정기지출 등록
- 고급 옵션 토글은 `고급 옵션 펼치기` / `고급 옵션 접기` 두 상태로 표시한다.

### 최근 기록 기반 자동 입력 보조

- 최근 기록 10개를 빠른 입력 영역으로 보여 준다.
- 최근 기록을 누르면 금액, 카테고리, 결제수단이 입력 폼에 채워진다.
- 캘린더에서 선택한 날짜는 지출 저장 날짜의 기본값으로 사용한다.

### 일반 지출 관리

- 일반 지출 기록 목록에서 수정할 수 있다.
- 일반 지출 기록 목록에서 삭제할 수 있다.
- 수정 중에는 지출 기록 수정 중 상태를 표시한다.
- 수정 저장 시 기존 지출 기록의 id를 유지한다.

### 정기지출 관리

- 정기지출 항목을 수정할 수 있다.
- 정기지출 항목을 삭제할 수 있다.
- 수정 중에는 현재 항목의 이름과 금액을 표시한다.

## 8. 분석 리포트

- 월 지출 요약
- 카테고리 비중
- 가장 많이 쓴 항목
- 간단한 절약 힌트

## 9. UI 구조 원칙

- 한 화면에 하나의 목적만 둔다.
- 정보는 요약 -> 핵심 -> 상세 순서로 배치한다.
- 색상 + 텍스트로 상태를 전달한다.
- 버튼 최소 높이 48px 유지
- 텍스트 최소 16px 이상
- 모바일 우선 설계
- Toss + Banksalad 하이브리드 톤을 유지한다.
- 카드 구조는 최소화하고, 핵심 숫자를 먼저 보여준다.
- 캘린더는 모바일에서 기본적으로 월별 지출 리스트를 먼저 보여 주고, 달력은 접기/펼치기 가능한 보조 영역으로 둔다.
- 캘린더의 금액 축약 표시는 한국식 `만` 단위를 사용한다.
- 기능 토글은 스위치 행 형태로 표시한다.
